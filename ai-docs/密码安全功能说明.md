# 密码安全功能说明

## 功能概述

系统已实现完善的密码安全机制，包括：
- ✅ BCrypt加密存储
- ✅ 弱密码检测
- ✅ 密码强度评估
- ✅ 密码修改功能
- ✅ 密码重置功能（管理员）

## 密码强度等级

| 等级 | 描述 | 要求 |
|------|------|------|
| 弱 | 不安全 | 纯数字、纯字母、常见弱密码 |
| 中等 | 基本安全 | 6位以上，包含字母和数字 |
| 强 | 安全 | 8位以上，包含大小写字母和数字 |
| 很强 | 非常安全 | 12位以上，包含大小写字母、数字和特殊字符 |

## 密码验证规则

### 宽松模式（默认）
- 最小长度：6位
- 最大长度：32位
- 必须包含：字母和数字
- 弱密码检测：是
- 强度提示：是

### 严格模式（可选）
- 最小长度：8位
- 最大长度：32位
- 必须包含：大小写字母、数字、特殊字符
- 弱密码检测：是
- 强度提示：是

## 弱密码检测

系统会自动检测以下类型的弱密码：

### 1. 常见弱密码
- 123456, password, 123456789, 12345678
- admin, letmein, welcome, qwerty
- 等30+个常见弱密码

### 2. 纯数字密码
- 例如：123456, 888888, 000000

### 3. 纯字母密码
- 例如：abcdef, password

### 4. 键盘序列
- qwerty, asdfgh, zxcvbn
- qazwsx, 123456, abcdef

### 5. 连续字符
- 连续递增：abc, 123, xyz
- 连续递减：cba, 321, zyx
- 重复字符：aaa, 111, xxx

## API接口

### 1. 修改密码

**接口**: `POST /api/auth/change-password`

**请求头**:
```
Authorization: Bearer {token}
X-User-Id: {userId}
```

**请求体**:
```json
{
  "oldPassword": "旧密码",
  "newPassword": "新密码",
  "confirmPassword": "确认新密码"
}
```

**响应**:
```json
{
  "success": true,
  "message": "密码修改成功",
  "data": null
}
```

**错误示例**:
```json
{
  "success": false,
  "message": "密码强度不足: 密码必须同时包含字母和数字; 密码过于简单，建议使用更复杂的密码",
  "data": null
}
```

### 2. 检查密码强度

**接口**: `POST /api/auth/check-password-strength`

**请求体**: 

## 密码加密

### BCrypt算法
- 使用BCrypt算法加密密码
- 自动生成盐值（salt）
- 每次加密结果不同，但验证结果一致
- 加密后的密码格式：`$2a$10$...`

### 示例
```
原始密码: myPassword123
加密后: $2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy
```

### 兼容性
系统支持明文密码和加密密码混合存在：
- 加密密码：使用BCrypt验证
- 明文密码：使用字符串比较（向后兼容）
- 建议：所有密码应迁移到加密存储

## 使用示例

### 前端示例（Vue 3）

#### 修改密码组件
```vue
<template>
  <el-form :model="form" :rules="rules" ref="formRef">
    <el-form-item label="旧密码" prop="oldPassword">
      <el-input v-model="form.oldPassword" type="password" />
    </el-form-item>
    
    <el-form-item label="新密码" prop="newPassword">
      <el-input 
        v-model="form.newPassword" 
        type="password"
        @input="checkStrength" 
      />
      <div class="strength-indicator">
        强度: {{ strength.strength }}
        <el-progress 
          :percentage="strength.level * 25" 
          :color="getStrengthColor(strength.level)"
        />
      </div>
      <div v-if="strength.suggestions.length" class="suggestions">
        <div v-for="(s, i) in strength.suggestions" :key="i">
          {{ s }}
        </div>
      </div>
    </el-form-item>
    
    <el-form-item label="确认密码" prop="confirmPassword">
      <el-input v-model="form.confirmPassword" type="password" />
    </el-form-item>
    
    <el-form-item>
      <el-button type="primary" @click="submit">修改密码</el-button>
    </el-form-item>
  </el-form>
</template>

<script setup>
import { ref, reactive } from 'vue'
import { ElMessage } from 'element-plus'
import { changePasswordApi, checkPasswordStrengthApi } from '@/api/auth'

const formRef = ref()
const form = reactive({
  oldPassword: '',
  newPassword: '',
  confirmPassword: ''
})

const strength = reactive({
  strength: '',
  level: 0,
  isWeak: false,
  suggestions: []
})

const rules = {
  oldPassword: [
    { required: true, message: '请输入旧密码', trigger: 'blur' }
  ],
  newPassword: [
    { required: true, message: '请输入新密码', trigger: 'blur' },
    { min: 6, message: '密码长度至少6位', trigger: 'blur' }
  ],
  confirmPassword: [
    { required: true, message: '请确认新密码', trigger: 'blur' },
    { 
      validator: (rule, value, callback) => {
        if (value !== form.newPassword) {
          callback(new Error('两次输入的密码不一致'))
        } else {
          callback()
        }
      }, 
      trigger: 'blur' 
    }
  ]
}

async function checkStrength() {
  if (form.newPassword.length < 6) return
  
  try {
    const result = await checkPasswordStrengthApi(form.newPassword)
    Object.assign(strength, result)
    
    if (strength.isWeak) {
      ElMessage.warning('密码强度较弱，建议使用更复杂的密码')
    }
  } catch (error) {
    console.error('检查密码强度失败:', error)
  }
}

async function submit() {
  await formRef.value.validate()
  
  try {
    await changePasswordApi(form)
    ElMessage.success('密码修改成功，请重新登录')
    // 跳转到登录页
  } catch (error) {
    ElMessage.error(error.message || '密码修改失败')
  }
}

function getStrengthColor(level) {
  const colors = ['#f56c6c', '#e6a23c', '#67c23a', '#409eff']
  return colors[level - 1] || '#909399'
}
</script>

<style scoped>

  margin-top: 8px;
  font-size: 12px;
  color: #606266;
}

.suggestions {
  margin-top: 8px;
  font-size: 12px;
  color: #e6a23c;
}
</style>
```

## 密码迁移

### 从明文迁移到加密

如果数据库中存在明文密码，可以使用以下脚本迁移：

```java
@Service
public class PasswordMigrationService {
    private final SysUserRepository userRepository;
    private final BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder();
    
    public void migratePasswords() {
        List<SysUser> users = userRepository.findAll();
        
        for (SysUser user : users) {
            String password = user.getPassword();
            
            // 检查是否已加密
            if (password != null && 
                !password.startsWith("$2a$") && 
                !password.startsWith("$2b$") && 
                !password.startsWith("$2y$")) {
                
                // 加密明文密码
                String encrypted = passwordEncoder.encode(password);
                user.setPassword(encrypted);
                userRepository.save(user);
                
                System.out.println("Migrated password for user: " + user.getUsername());
            }
        }
    }
}
```

## 常见问题

### Q1: 为什么我的密码被拒绝？
A: 密码可能过于简单或在弱密码列表中。请使用至少6位，包含字母和数字的密码。

### Q2: 如何提高密码强度？
A: 
- 增加密码长度（建议8位以上）
- 混合使用大小写字母
- 添加数字和特殊字符
- 避免使用常见单词和序列

### Q3: 忘记密码怎么办？
A: 联系管理员重置密码。管理员可以使用重置密码功能为您设置新密码。

### Q4: 密码加密后还能恢复吗？
A: 不能。BCrypt是单向加密算法，加密后无法解密。只能重置密码。

### Q5: 系统如何存储密码？
A: 系统使用BCrypt算法加密存储密码，不存储明文密码。即使数据库泄露，攻击者也无法获取原始密码。

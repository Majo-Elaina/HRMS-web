# 亚托莉助手 - 完整配置与部署指南

## 一、功能概述

亚托莉AI助手是HRMS系统的智能聊天助手，具备以下核心能力：

- **动态数据获取**：实时从数据库读取最新数据，无需重启服务
- **聊天记录持久化**：支持用户切换和页面刷新后保持聊天历史
- **全系统只读问答**：覆盖员工、部门、考勤、薪资、审批等所有业务域
- **权限边界控制**：严格按照用户权限返回数据，保护敏感信息
- **多模型支持**：支持Ollama本地模型和第三方API切换
- **天气查询**：集成高德地图天气API，支持全国主要城市天气查询
- **连接稳定性**：自动重试机制，确保模型连接稳定可靠
- **现代化UI**：思考状态动画、消息来源提示、建议问题等

## 二、AI模型配置

### 2.1 配置文件说明

配置文件位置：`src/main/resources/application.properties`

```properties
# AI助手基础配置
ai.chat.assistant-name=亚托莉
ai.chat.active-provider=dashscope-qwen
ai.chat.expose-provider-errors=false

# Ollama本地模型配置
ai.chat.providers[0].name=ollama-local
ai.chat.providers[0].type=ollama
ai.chat.providers[0].enabled=${AI_CHAT_PROVIDER_OLLAMA_ENABLED:true}
ai.chat.providers[0].base-url=${AI_CHAT_PROVIDER_OLLAMA_BASE_URL:http://127.0.0.1:11434}
ai.chat.providers[0].model=${AI_CHAT_PROVIDER_OLLAMA_MODEL:qwen3:4b}
ai.chat.providers[0].temperature=${AI_CHAT_PROVIDER_OLLAMA_TEMPERATURE:0.7}

# OpenAI兼容API配置（阿里云百炼）
ai.chat.providers[1].name=dashscope-qwen
ai.chat.providers[1].type=openai-compatible
ai.chat.providers[1].enabled=${AI_CHAT_PROVIDER_OPENAI_ENABLED:true}
ai.chat.providers[1].base-url=${AI_CHAT_PROVIDER_OPENAI_BASE_URL:https://dashscope.aliyuncs.com/compatible-mode/v1}
ai.chat.providers[1].api-key=${AI_CHAT_PROVIDER_OPENAI_API_KEY:}
ai.chat.providers[1].model=${AI_CHAT_PROVIDER_OPENAI_MODEL:qwen-plus}
ai.chat.providers[1].temperature=${AI_CHAT_PROVIDER_OPENAI_TEMPERATURE:0.7}
```

### 2.2 环境变量配置

推荐使用环境变量管理敏感信息：

```bash
# 设置活跃的Provider
export AI_CHAT_ACTIVE_PROVIDER=dashscope-qwen

# 阿里云百炼API Key

export AI_CHAT_PROVIDER_OLLAMA_ENABLED=true
export AI_CHAT_PROVIDER_OLLAMA_BASE_URL=http://127.0.0.1:11434
export AI_CHAT_PROVIDER_OLLAMA_MODEL=qwen3:4b
```

### 2.3 支持的模型类型

#### Ollama本地模型

优点：
- 完全本地部署，数据不出网
- 无API调用费用
- 响应速度快（有GPU时）

推荐模型：
- `qwen3:4b`：轻量级，适合开发测试
- `qwen3:14b`：平衡性能和资源
- `deepseek-r1:7b`：推理能力强

安装方法：
```bash
# 安装Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 下载模型
ollama pull qwen3:4b
```

#### OpenAI兼容API

支持的服务商：
- 阿里云百炼（推荐）
- OpenAI官方
- DeepSeek
- 其他兼容OpenAI格式的服务

推荐模型：
- `qwen-plus`：性能均衡，价格适中
- `qwen-max`：最强性能
- `qwen-turbo`：快速响应

### 2.4 连接稳定性优化

系统已实现以下优化措施：

#### 自动重试机制
- 最多重试3次
- 使用指数退避策略（1秒、2秒、3秒）
- 只有在所有重试都失败后才返回错误

#### 超时配置
- 连接超时：10秒
- 读取超时：30秒
- 确保网络波动时有足够的时间完成请求

#### 优雅降级
当模型不可用时，系统会：
1. 使用只读数据构建结构化回答
2. 保持亚托莉的角色设定
3. 提供友好的错误提示
4. 记录模型状态供前端显示

配置类：`RestClientConfig.java`
```java
@Bean
public RestClient.Builder restClientBuilder() {
    ClientHttpRequestFactorySettings settings = ClientHttpRequestFactorySettings.DEFAULTS
            .withConnectTimeout(Duration.ofSeconds(10))
            .withReadTimeout(Duration.ofSeconds(30));
    
    ClientHttpRequestFactory requestFactory = ClientHttpRequestFactories.get(settings);
    
    return RestClient.builder()
            .requestFactory(requestFactory);
}
```

## 三、天气服务配置

### 3.1 功能说明

亚托莉现在可以回答天气相关的问题，例如：
- "今天绵阳天气怎么样？"
- "成都的天气如何？"
- "北京现在的气温是多少？"

### 3.2 申请高德地图API Key（免费）

1. 访问高德开放平台：https://lbs.amap.com/
2. 注册/登录账号
3. 进入控制台 -> 应用管理 -> 我的应用
4. 创建新应用，添加Key
5. 服务平台选择：Web服务
6. 复制生成的Key

### 3.3 配置到项目

在 `application.properties` 中设置：

```properties
# 天气服务配置（高德地图API）
weather.amap.key=${WEATHER_AMAP_KEY:your_amap_key_here}
```

或者通过环境变量设置：

```bash
export WEATHER_AMAP_KEY=你的高德API_Key
```

### 3.4 使用限制

- 高德地图个人开发者账号：每天100万次调用（完全够用）
- 支持全国主要城市的实时天气查询
- 如果不配置Key，亚托莉会提示"天气服务暂未配置"

### 3.5 支持的城市

系统自动识别用户消息中的城市名称，支持的城市包括：

- 直辖市：北京、上海、天津、重庆
- 省会城市：成都、绵阳、广州、深圳、杭州、南京、武汉、西安等
- 主要城市：苏州、青岛、大连、厦门、昆明等

默认城市：绵阳

## 五、架构设计

### 5.1 推荐总体架构

```text
浏览器
  -> HRMS 前端
  -> HRMS 后端 /api/ai/chat
  -> 可切换的模型 Provider
  -> 数据库只读知识层
```

关键原则：
- 浏览器永远只访问 HRMS 后端
- 浏览器不直接访问模型服务
- 模型调用统一由后端完成
- 系统数据统一由后端只读工具层提供
- 人设约束和业务事实必须分开管理

### 5.2 核心组件

#### AiChatService
- 聊天服务协调层
- 多模型支持和切换
- 角色人设管理
- 聊天记录持久化
- 优雅降级处理

#### AiReadonlyKnowledgeService
- 只读知识服务层
- 问题分类识别
- 实时数据查询
- 权限范围过滤
- 结构化上下文构建

#### WeatherService
- 天气服务层
- 高德地图API集成
- 城市识别和查询
- 天气数据格式化

#### RestClientConfig
- HTTP客户端配置
- 超时和重试设置
- 连接池管理

### 5.3 数据流程

1. 用户在前端输入问题
2. 前端发送请求到 `/api/ai/chat`
3. 后端进行权限校验
4. 只读知识层根据问题匹配业务域并读取数据
5. 如果涉及天气，调用天气服务
6. 角色人设 + 系统事实上下文 + 对话历史一起交给模型
7. 模型按亚托莉口吻生成回答
8. 保存对话记录到数据库
9. 返回响应给前端

## 六、权限控制

### 6.1 权限边界

亚托莉虽然能回答更多问题，但不是无边界读取。

系统当前遵循：
- 普通员工默认优先读自己的数据
- 高敏感域按现有角色权限控制
- 报表、审批规则、角色权限、模块范围、部门模板等仍会校验查看权限
- 模型只能读取，不允许写入业务数据

### 6.2 数据范围控制

- **普通员工**：主要查看自己相关的数据
- **部门经理**：可查看本部门员工数据
- **HR人员**：可查看全公司人事数据
- **财务人员**：可查看薪资相关数据
- **管理员**：可查看所有系统数据

## 七、常见问题

### Q1: 亚托莉的回答是动态调整的吗？

**A: 是的，完全动态。** 每次回答都会实时查询数据库最新数据，无需重启或修改代码。

### Q2: 改了角色、身份标签、审批规则等，需要修改代码吗？

**A: 不需要。** 亚托莉助手设计为完全数据驱动，所有业务规则变更通过数据库配置即可生效。

### Q3: 如何确保数据安全？

**A: 多层权限控制：**
- 用户身份验证
- 角色权限检查
- 只读数据限制
- 敏感信息过滤

### Q4: 模型连接经常失败怎么办？

**A: 系统已实现自动重试机制：**
- 最多重试3次
- 使用指数退避策略
- 即使网络波动也能成功连接
- 如果仍然失败，检查网络和API配置

### Q5: 天气服务不可用怎么办？

**A: 检查以下几点：**
1. 是否配置了高德API Key
2. API Key是否有效
3. 是否超过每日调用限制
4. 网络是否能访问高德服务

### Q6: 如何切换不同的AI模型？

**A: 修改配置文件：**
```properties
# 切换到Ollama本地模型
ai.chat.active-provider=ollama-local

# 切换到阿里云百炼
ai.chat.active-provider=dashscope-qwen
```

重启服务后生效。

### Q7: 聊天记录会保存多久？

**A: 永久保存。** 聊天记录存储在数据库中，除非用户主动清空，否则会一直保留。

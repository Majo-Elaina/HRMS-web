# HRMS 接口文档

本文档统一维护 HRMS 项目的所有后端接口说明，覆盖前端已接通的主要业务接口。

## 通用说明

### 响应结构

所有接口统一返回以下格式：

```json
{
  "success": true,
  "message": "ok",
  "data": {}
}
```

分页响应 `data` 结构：

```json
{
  "items": [],
  "total": 0,
  "page": 1,
  "size": 10
}
```

### 服务地址

- 后端默认地址：`http://localhost:8080`
- 前端开发代理：`/api -> http://localhost:8080`

## 一、认证与权限管理

### 1.1 用户认证

#### POST /api/auth/login
用途：账号登录，返回 token 和当前用户资料。

请求示例：
```json
{
  "username": "emp_zhou",
  "password": "123456"
}
```

响应示例：
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "userId": 1,
    "username": "emp_zhou",
    "empId": 1,
    "empName": "周杰伦",
    "deptId": 1,
    "deptName": "技术部",
    "positionId": 1,
    "positionName": "高级工程师",
    "roleId": 2,
    "roleName": "普通员工",
    "roleCode": "EMPLOYEE",
    "identityTagCode": "EMPLOYEE",
    "permissions": ["base:employee:view", "attendance:record:view"]
  }
}
```

#### GET /api/auth/profile/{userId}
用途：刷新登录用户资料与权限上下文。

### 1.4 角色权限关系

#### GET /api/role-permissions/role/{roleId}
用途：查询角色权限关系明细。

#### GET /api/role-permissions/role/{roleId}/perm-ids
用途：查询角色已分配权限 ID 列表，用于权限配置弹窗回显。

响应示例：
```json
[1, 2, 3, 30]
```

#### GET /api/role-permissions/role/{roleId}/permissions
用途：查询角色权限实体列表。

#### PUT /api/role-permissions/role/{roleId}
用途：整组替换角色权限。

请求示例：
```json
{
  "permIds": [1, 2, 3, 30, 43]
}
```

处理逻辑：
1. 删除当前角色旧的 `role_permission`
2. 立即 flush
3. 批量写入新的 `perm_id`

用途：查询单个请假单详情。

用途：新增请假申请。

#### PUT /api/leave-requests/{id}
用途：更新请假单（审批、取消等操作）。

常见更新字段：
- `status`：单据状态
- `approverId`：审批人ID
- `pendingApproverTag`：当前待审批人标签
- `pendingApproverScope`：当前待审批人范围
- `nextApproverTag`：下一级审批人标签
- `nextApproverScope`：下一级审批人范围
- `approveRemark`：审批备注

#### DELETE /api/leave-requests/{id}
用途：删除请假单。

## 五、AI助手接口

### 5.1 聊天接口

#### POST /api/ai/chat
用途：与亚托莉AI助手进行对话。

请求示例：
```json
{
  "userId": 1,
  "message": "今天绵阳的天气怎么样？",
  "history": [
    { "role": "user", "content": "你好" },
    { "role": "assistant", "content": "我是亚托莉，请把问题交给我吧。" }
  ]
}
```

响应示例：
```json
{
  "assistantName": "亚托莉",
  "provider": "dashscope-qwen",
  "model": "qwen-plus",
  "providerAvailable": true,
  "notice": "",
  "reply": "主人，我刚刚查询了绵阳的天气信息...",
  "usedSystemData": true,
  "sources": [
    { "type": "weather.api", "title": "天气数据" }
  ]
}
```

字段说明：
- `providerAvailable`：当前模型 provider 是否可用
- `notice`：provider 不可用或切换提醒
- `usedSystemData`：这轮回答是否读取了系统只读数据
- `sources`：这轮实际命中的系统数据来源

### 5.2 聊天历史

#### GET /api/ai/history/{userId}
用途：获取用户的聊天历史记录。

响应示例：
```json
[
  {
    "id": 1,
    "role": "user",
    "content": "你好",
    "providerName": "dashscope-qwen",
    "modelName": "qwen-plus",
    "usedSystemData": false,
    "createTime": "2026-03-24T10:30:00"
  }
]
```

#### DELETE /api/ai/history/{userId}
用途：清空用户的聊天历史记录。

### 5.3 Provider信息

#### GET /api/ai/provider-info
用途：获取当前AI模型配置信息。

响应示例：
```json
{
  "assistantName": "亚托莉",
  "provider": "dashscope-qwen",
  "model": "qwen-plus",
  "providerAvailable": true
}
```

## 附录：审批规则说明

### 请假审批规则

当前系统统一采用以下请假审批链：

- 普通员工请假：`HR专员 -> HR经理`
- 部门经理请假：`HR经理 -> 管理员`
- HR经理请假：`管理员`

### 薪资审批规则

- 薪资记录审批：`财务经理`
- 薪资配置审批：`财务经理`

### 数据库规则表

`approval_rule.type_code = 'leave'`

| applicant_tag | days_op | days_value | first_approver_tag | second_approver_tag | second_approver_scope |
| --- | --- | --- | --- | --- | --- |
| EMPLOYEE | <= | 3 | HR_SPECIALIST | HR_MANAGER | company |
| EMPLOYEE | > | 3 | HR_SPECIALIST | HR_MANAGER | company |
| MANAGER | any | 0 | HR_MANAGER | ADMIN | company |
| HR_MANAGER | any | 0 | ADMIN | 空 | company |

`approval_rule.type_code = 'salary_record'`

| applicant_tag | first_approver_tag |
| --- | --- |
| FINANCE_SPECIALIST | FINANCE_MANAGER |

`approval_rule.type_code = 'salary_config'`

| applicant_tag | first_approver_tag |
| --- | --- |
| FINANCE_SPECIALIST | FINANCE_MANAGER |
